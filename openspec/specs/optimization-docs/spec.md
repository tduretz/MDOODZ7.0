## ADDED Requirements

### Requirement: OPTIMIZATIONS.md at repository root
The system SHALL include an `OPTIMIZATIONS.md` file at the repository root that documents the performance optimisations, validation results, and usage guidance.

#### Scenario: File exists and is accessible
- **WHEN** a user navigates to the repository root
- **THEN** they SHALL find `OPTIMIZATIONS.md` alongside `README.md`

### Requirement: Overview of optimisations
The document SHALL contain a section explaining what the two optimisations are: the PCG thermal solver (`thermal_solver = 1`) and the fused P2Mastah interpolation (`interp_mode = 3`), with a brief description of how each works.

#### Scenario: PCG solver described
- **WHEN** a user reads the overview
- **THEN** they SHALL understand that the PCG solver replaces CHOLMOD for the thermal equation, uses Jacobi preconditioning, and benefits from warm-starting across timesteps

#### Scenario: Fused P2Mastah described
- **WHEN** a user reads the overview
- **THEN** they SHALL understand that interp_mode=3 fuses multiple particle-to-grid interpolation passes into a single loop, reducing memory traffic

### Requirement: Performance results summary
The document SHALL contain a summary table of benchmark results from Experiments 1–8, showing wall-time reduction, per-phase breakdown, and thread scaling behaviour.

#### Scenario: Large-grid results presented
- **WHEN** a user reads the performance section
- **THEN** they SHALL find the cumulative wall-time reduction (59% at 16 threads on 601×401) and per-phase improvements (thermal 98%, interp 66%)

#### Scenario: Small-grid results presented
- **WHEN** a user reads the performance section
- **THEN** they SHALL find the Experiment 8 finding that optimisations have zero measurable effect on grids below ~100×100

### Requirement: Validation evidence with figures
The document SHALL embed or reference the validation figures (PCG convergence plot, L2 error bar chart, interp difference heatmap) generated by the visual tests, with brief interpretation.

#### Scenario: Figures referenced
- **WHEN** a user reads the validation section
- **THEN** they SHALL see references to the three validation figures with captions explaining what each demonstrates

#### Scenario: L2 error interpretation
- **WHEN** a user reads the validation section
- **THEN** they SHALL understand that CHOLMOD and PCG produce the same L2 error against analytic solutions, confirming correctness

### Requirement: Usage instructions
The document SHALL explain how to enable each optimisation via `.txt` parameter file settings.

#### Scenario: Enabling PCG solver
- **WHEN** a user reads the usage section
- **THEN** they SHALL find instructions to set `thermal_solver = 1`, `max_its_thermal`, and `rel_tol_thermal`

#### Scenario: Enabling fused interpolation
- **WHEN** a user reads the usage section
- **THEN** they SHALL find instructions to set `interp_mode = 3`

### Requirement: Guidance on when NOT to use optimisations
The document SHALL contain a section explaining when the optimisations provide no benefit or may be counterproductive.

#### Scenario: Small grid warning
- **WHEN** a user reads the guidance section
- **THEN** they SHALL learn that for grids below ~100×100, the optimisations have zero measurable effect (Experiment 8) and the defaults (`thermal_solver = 0`, `interp_mode = 0`) are fine

#### Scenario: Single-thread advisory
- **WHEN** a user reads the guidance section
- **THEN** they SHALL learn that PCG benefits are greatest at high thread counts (4+ threads) and negligible at 1 thread where CHOLMOD is already fast

#### Scenario: CHOLMOD fallback note
- **WHEN** a user reads the guidance section
- **THEN** they SHALL understand that if PCG fails to converge the system automatically falls back to CHOLMOD, so enabling `thermal_solver = 1` is always safe
