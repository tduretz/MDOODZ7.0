## Context

The ThermoElastic benchmark validates thermal pressurisation: confined thermal expansion produces pressure rise $\Delta P = K \alpha \Delta T$ where $K = 1/\beta = 10$ GPa, $\alpha = 10^{-5}$ K$^{-1}$. Both the CI test (GTest in `ThermalVerificationTests.cpp`) and visual test (`PlotThermoElastic()` in `Validation.cpp`) already exist. The CI test also compares CHOLMOD vs PCG thermal solvers. However:

1. The current gnuplot (`ThermoElastic.gnu`) plots T(t) and P(t) as separate panels. The desired plot is **P vs T** with an analytical overlay and a **pressure error panel** beneath — matching the Julia `:PTtime` visualisation.
2. The thermal validation plots (`thermoelastic.png`, `oceanic_cooling.png`, `thermal_accuracy.png`) exist in `VISUAL_TESTS/img/` but are not embedded in the README.
3. The CI test already has the CHOLMOD-vs-PCG cross-comparison — no new CI test is needed.

## Goals / Non-Goals

**Goals:**
- Replace the ThermoElastic gnuplot with a P-vs-T + error two-panel layout
- Update `PlotThermoElastic()` data output to include error column for the new plot
- Embed thermoelastic, oceanic cooling, and thermal accuracy plots in `VISUAL_TESTS/readme.md`

**Non-Goals:**
- Adding new CI tests (the CHOLMOD-vs-PCG comparison already exists in `ThermoElasticPCG`)
- Changing the number of simulation steps (keep 5 steps — sufficient for the visual test)
- Modifying the ThermoElastic scenario physics or parameters
- Changing the OceanicCooling or ThermalAccuracy plotting code

## Decisions

### 1. P-vs-T plot layout instead of separate time-series panels

The current ThermoElastic.gnu shows T(t) and P(t) as two side-by-side panels. Replace with:
- **Top panel**: P vs T scatter (CHOLMOD + PCG) with analytical line $P = -K \alpha (T - T_0)$
- **Bottom panel**: Relative pressure error (%) vs step for both solvers

This matches the Julia `:PTtime` visualisation and directly shows the linear relationship between pressure and temperature, which is the physics being validated.

### 2. Compute error in C++ and write to data file

Add a `P_error_cholmod` and `P_error_pcg` column to `thermoelastic.dat` so the gnuplot script stays simple. The analytical pressure is already computed in `PlotThermoElastic()`.

### 3. Embed all three thermal plots in README

Add three sections after the existing Shrinking/TopoBench entries: ThermoElastic, OceanicCooling, ThermalAccuracy. Each gets a brief physics description and an image embed. No reference images needed — the analytical solution is overlaid on the plot itself.

## Risks / Trade-offs

- [Sign convention] The visual test uses $P_\text{ana} = -K \alpha \Delta T$ (compressive = negative) while the CI test uses $P_\text{ana} = +K \alpha \Delta T$. Both are correct but plot sign may differ. → Verify in `PlotThermoElastic()` and keep consistent with existing code.
- [Image staleness] README images are committed PNGs generated by `make run-vis`. If the simulation changes, images must be regenerated. → This is the existing pattern for all visual tests, no change needed.
