## 2. Snapshots chosen — exact mapping to "~5 Ma"

Because `dt` is adaptive, the index does not map 1-to-1 to model
time across the three runs. We pick step indices that put all three
runs within ~26 kyr of each other near $t = 5\,\mathrm{Ma}$ — the
three quoted below differ by 0.5 % in model time, tight enough that
the panels are directly comparable kinematically:

| Panel  | Snapshot                  | Model time (s)              | Model time (Ma) |

| Right  | `Output00640.gzip.h5`    | 1.5827 × 10¹⁴              | 5.015          |

(Time is in `/Model/Params[0]`; verified in

## 4. "Lithospheric-mantle" statistics — exact definition

A cell is counted as lithospheric mantle iff
`/VizGrid/compo == 2` (this is the phase-2 mask written by
MDOODZ; phases 3 / 4 / 5 are asthenosphere / sediment-1 / sediment-2
respectively in the RiftingRoman setup, see the `.txt` files in §1).

The three statistics quoted in summary §3.2 are computed as:

```python
mask = (compo == 2)
delta_max  = ani_fac[mask].max()
median_d   = np.median(ani_fac[mask])
frac_gt4   = (ani_fac[mask] > 4.0).mean() * 100  # %
```

Reproducible numbers (printed by `plot_07_rifting_comparison.py`
when run with `--print-stats`, regenerated 2026-05-15):

| Panel  | t (Ma) | δ range        | median δ | % cells > 4 | N (cells) |
|--------|-------:|----------------|---------:|------------:|----------:|
| Left   | 5.041  | [1.05, 4.00]   | 4.00     | 0.0         | 15 988    |
| Middle | 5.027  | [1.00, 13.91]  | 1.17     | 13.0        | 15 948    |
| Right  | 5.015  | [1.00, 13.27]  | 4.00     | 48.2        | 16 035    |

Notes:
- "Left" reads 0 % > 4 because the const-aniso config hard-caps δ at
  `aniso_factor = 4`; the maximum is reached but never exceeded.
- "Middle" median sits at 1.17 because the cold, undeformed bulk
  dominates the lith-mantle mask, with the high-δ shear bands a
  small minority by cell count.
- "Right" median sits at 4.00 because the inherited F-init carries
  δ = 4 throughout the cold lith-mantle bulk; only the warm rift
  edges (gate-released or thermally transmuted) drop to ≈ 1.

If these numbers drift after a re-run, propagate the new values to
both this table and summary §3.2 in
[`CALIBRATION_AND_DYNAMIC_FSTRAIN.md`](../CALIBRATION_AND_DYNAMIC_FSTRAIN.md).

## 5. Regenerating the figure

```bash
cd <repo>
python3 VISUAL_TESTS/aniso_fstrain/plot_07_rifting_comparison.py \
    --const cmake-exec/RiftingRoman/run_constaniso/Output00640.gzip.h5 \
    --init1 cmake-exec/RiftingRoman/run_aniso3_init1/Output00720.gzip.h5 \
    --v3    cmake-exec/RiftingRoman/run_aniso3_v3/Output00640.gzip.h5 \
    --out   VISUAL_TESTS/img/aniso_fstrain/07_rifting_comparison_5Ma.png \
    --print-stats
```

If any of the three runs is re-driven and the closest-to-5-Ma index
shifts, the script's `--print-stats` output is what should be
checked against this provenance file. Update both the table here
and the corresponding numbers in summary §3.2 if so.
